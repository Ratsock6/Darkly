Faille identifiée : Information Disclosure
Page : /?page=recover

Description:
	Sur le site, certaines informations sensibles (ici l’adresse e-mail) ne sont pas affichées directement à l’écran,
	mais elles sont tout de même présentes en clair dans le code source HTML de la page, simplement masquées avec l’attribut hidden.

	Cela constitue une faille de type Information Disclosure / Sensitive Data Exposure, car :

	- La donnée est transmise côté client alors qu’elle ne devrait pas l’être.
	- Toute personne peut la récupérer facilement via un clic droit → Inspecter le code source.
	- La donnée sensible est donc exposée, même si elle n’apparaît pas dans l’interface du site.
	- L'information peut etre modifiée par l'utilisateur.

Exploitation :
	En inspectant le code source de la page, nous avons pu récupérer l’adresse e-mail de l’administrateur du site, qui est "webmaster@borntosec.com"
	cette adresse e-mail est envoyée dans le formulaire de récupération de mot de passe.
	Nous avons donc modifié cette adresse e-mail par la nôtre "dan@borntosec.com",
	ce qui a affiché le flag.

Correctifs :
	Ne pas stocker l'adresse e-mail de l'administrateur dans le code HTML mais l'intégrer côté serveur.
	Ou plus généralement, ne pas stocker d'informations sensibles côté client, même dans des champs masqués.
